## Decisions

### D1: 三附录命名规范 — `appendix-{a,b,c}-{glossary,references,examples}.md`

**决策**:使用 `appendix-{letter}-{topic}.md` 格式,字母按 a→b→c 顺序表示附录编号。

**理由**:
- 与已有编号 01-27 区分(主文档 + 附录双编号体系)
- 字母顺序明确(A < B < C),读者可按目录顺序阅读
- 文件名包含主题词,IDE 搜索友好
- 与 tasks.md / SNAPSHOT.md 已有"A/B/C 附录"命名一致

**备选方案**:
- `28-glossary.md` 等数字编号 → 失去"附录"语义暗示
- `glossary.md` 等纯名 → 失去项目编号体系一致性

### D2: 术语表按 A-Z 字母索引,每条带分类标签

**决策**:每条术语用 `**术语名** (分类)` 格式,如 `**WFE** (M6.3 / ARM / 指令)`,1-2 句定义,可选附"M1-M7 引用位置"。

**理由**:
- 字母索引便于快速查找(读者已知术语名)
- 分类标签帮助读者判断"该术语在我关心的主题吗"
- 引用 M1-M7 节号让读者"快速跳转到详细解释"
- 1-2 句定义保持"索引型"简洁性(详细解释在 M1-M7)

**备选方案**:
- 按 M1-M7 主题分类 → 重复术语跨多主题,反而混乱
- 纯定义无引用 → 失去与 M1-M7 链接,信息孤岛

### D3: 参考资料按 5 大类组织

**决策**:5 大类 = Embassy 官方 / ARM & Cortex-M 规范 / Rust 资源 / 工具链 / 学习社区。

**理由**:
- 读者查阅参考资料时,通常先确定"我需要哪类资源",分类加快定位
- 5 大类覆盖学习 embassy 必需的全部外部资源
- 与 M1-M7 已引用的外部资料保持一致(去重合并)

**备选方案**:
- 全部铺平为单列表 → 失去分类导航
- 按 M1-M7 主题分类 → 同一资源可能被多个主题引用,重复

### D4: 示例索引按"主题 → 平台 → 文件"三层分类

**决策**:每条示例用三层结构:`主题(Mx.x 节号) → 平台 → embassy/examples/<path> 文件路径 + 1 句用途`。

**理由**:
- 主题分类帮助读者"想找某类示例"时快速定位
- 平台分类帮助读者"我有这块板"时找到对应案例
- 文件路径直接给读者"如何运行"
- 引用 Mx.x 节号让读者跳到 M1-M7 详细分析

**备选方案**:
- 纯按平台分类 → 同一平台多个主题示例混在一起
- 纯按主题分类 → 失去平台过滤维度

### D5: 深度精简(各篇 400-600 行)

**决策**:每篇附录 400-600 行,深度灵活,无 Mermaid(附录非分析型)。

**理由**:
- 用户决策"看情况精简就好"
- 附录是"索引型"非"教程型",深度展开反而冗余
- 与 M1-M7 27 篇(平均 900 行)形成"主文档 + 索引"对照
- 0 Mermaid 合理(无流程图 / 时序图需求)

**备选方案**:
- 1000+ 行(与主文档持平)→ 重复 M1-M7 内容,价值低
- < 300 行 → 覆盖不全

### D6: 用 grep 从 M1-M7 提取术语与章节引用,不重新发散

**决策**:撰写时用 `grep -oE` 提取 M1-M7 中的"加粗术语"与"节标题"作为术语表 / 示例索引的来源。

**理由**:
- 术语表应覆盖 M1-M7 实际用过的术语,避免遗漏
- 章节引用应来自 M1-M7 实际节号,避免凭空捏造
- 双向校验:每个术语在 M1-M7 中有出处

**备选方案**:
- 仅凭记忆撰写 → 易遗漏,术语表不完整
- 重新独立组织 → 与 M1-M7 体系不一致

## Risks / Trade-offs

| 风险 | 影响 | 缓解 |
|------|------|------|
| 3 附录合计 ~1500 行,工作量大 | 容易出现"敷衍"内容 | 强制每术语带 M1-M7 引用,确保与主文档的链接密度 |
| 术语表覆盖广度 | 可能遗漏关键术语 | 用 grep 自动提取 M1-M7 加粗词 + 人工补充 embassy 通用术语 |
| 参考资料 URL 漂移 | 半年后链接失效 | 标注"参考日期: 2026-06",并提供 embassy 官方主页作为"入口" |
| 示例索引文件路径 | embassy 版本升级时变化 | 标注 embassy 版本(`0.5+`),提供仓库 URL 作为参考 |
| 跨链接维护成本 | M1-M7 章节号变化时需更新 | M1-M7 已收官,章节号稳定;链接密度可接受 |
| 0 Mermaid 风险 | 附录无图可能"视觉单调" | 大量表格 + 分类列表弥补;附录不需时序图 |
| 与 M1-M7 内容重复 | 索引型文档与教程型重复 | 严格"每条目 1-2 句"原则,详细解释跳到 M1-M7 |
| 三附录内部交叉 | A 引用 B, B 引用 C | 显式建立"附录交叉链接表",避免循环引用 |
| grep 提取可能误识别 | 注释 / 代码中的术语混入 | 仅 grep 文档正文(非代码块),人工复核 |
